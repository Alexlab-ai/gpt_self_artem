## 1. О ПРОЕКТЕ

GPT-SELF — бот-помощник для прохождения программы 12 шагов (АА/АН) с AI-поддержкой.

- **Платформа:** Telegram Bot + Backend API + (будущее) Desktop
- **Стек:** Python, FastAPI, aiogram, PostgreSQL, ChromaDB, OpenAI API
- **Деплой:** Render (Backend + Bot отдельно)
- **Репозиторий:** twelvesteps (backend) + twelvesteps_tgbot (bot)

## 4. ПРОМТЫ (ДЕЙСТВУЮЩИЕ)

### 4.1 SOS — «Не понимаю вопрос»

```
Пользователь нажал кнопку: "Не понимаю вопрос".

У тебя есть:
1. knowledge block по 12 шагам
2. текст текущего вопроса
3. номер текущего шага

Задача:
- Переформулируй вопрос доступно, коротко, по-человечески
- Объясни, про что этот вопрос в контексте шага
- Приведи 1–2 варианта, как его можно понять
- Не анализируй, не интерпретируй, не давай готовые ответы
```

### 4.2 SOS — «Помоги понять, куда смотреть»

```
Пользователь нажал SOS: "Помоги понять, куда смотреть".

Он говорит: "чувствую, что что-то есть — но не понимаю, где это".
Это не про воспоминание, а про фокус.

Задача:
- Помоги осознать, в какую область внимания посмотреть
- Не предлагай примеры, не генерируй истории, не трактуй
- Генерируй вопросы-направления, расширяющие угол обзора
- 5-6 направляющих вопросов
```

### 4.3 SOS — «Нужны примеры»

```
Сформируй:
1. 12–18 коротких бытовых ситуаций (1 строка) из разных сфер:
   семья, работа, здоровье, финансы, онлайн, отношения, быт, дети, сообщество.
2. 4 навигационные "крючка памяти" — куда посмотреть прямо сейчас
3. 5–7 ранних признаков (телесные, мыслительные, поведенческие)

Не предлагай готовые ответы. Не используй названия веществ. Не интерпретируй.
```

### 4.4 SOS — «Просто тяжело, нужна поддержка»

```
Задача: Поддержать. Мягко. Без анализа.
- Никаких вопросов «что произошло?»
- Не двигать, не вести, не "развивать тему"
- Просто быть рядом
- Простые фразы. Короткие. Человеческие.

Нельзя: развлечения, "всё будет хорошо", анализ, "применить шаг", интерпретации.
```

### 4.5 Промты, которые НУЖНО НАПИСАТЬ

- [ ] Обратная связь GPT после ответа на вопрос шага
- [ ] Обратная связь GPT после завершения самоанализа (10 шаг)
- [ ] Обратная связь GPT после благодарности

## 9. СЕМАНТИЧЕСКОЕ ЯДРО GPT-SELF

> **Подробная спецификация системы памяти и блоков: `docs/memory_blocks_spec.md`**

### 9.1 Классификация сообщений

Каждое сообщение анализируется через GPT:

- parts: разбиение на смысловые части
- emotion: определённая эмоция
- importance: 0-100
- blocks: блоки для памяти
- thinking_frame: когнитивный паттерн
- level_of_mind: уровень осознанности 0-100
- memory_type: volatile / dynamic / stable
- action: рекомендуемое действие
- strategy_hint: подсказка

### 9.2 Трёхуровневая память

|Уровень |Описание                  |Хранение  |
|--------|--------------------------|----------|
|Volatile|Текущий диалог (30-60 мин)|TTL в кэше|
|Dynamic |Повторяющиеся эпизоды     |Кэш + БД  |
|Stable  |Ядро личности             |PostgreSQL|

### 9.3 MetaSloy v1 (PRO)

Встроенный режим наблюдения — НЕ блок, а режим восприятия.

- Глобальное наблюдение сквозь все блоки
- Определяет фреймы: "контроль", "поиск", "откат", "зрелость", "наблюдатель"
- Сравнивает фреймы во времени
- Передаёт сигналы в GPT-SELF (не хранит сам)

### 9.4 Двухуровневый ответ (PRO)

```json
{
  "emotional": "Мне кажется, ты испытываешь растерянность...",
  "structural": "Судя по контексту, ты столкнулся с конфликтом..."
}
```

## 15. ФАЗА 10: КОЛЛЕКТИВНЫЙ ИНТЕЛЛЕКТ (после 500+ юзеров)

> **Не реализовывать до набора базы 500+ активных пользователей.**

### 15.1 Статистика прохождения (реализовать рано — на Фазе 5)

- Какие вопросы чаще ставят в тупик (длинная пауза, SOS)
- На каких шагах бросают
- Средний срок прохождения шага
- Просто счётчики в БД, без AI

### 15.2 Кнопка «Как другие отвечали на этот вопрос» (Premium/PRO)

- Анонимные, модерированные примеры ответов
- Юзер может дать согласие: «Разрешить использовать мой ответ анонимно»
- Показывать 2-3 примера, без имён, без деталей
- Реализация: отдельная таблица shared_answers (answer_id, step, question, anonymized_text, approved)

### 15.3 Агрегация паттернов

- На основе блоковой системы: «на шаге 4 чаще всего: семья + вина + стыд»
- Для улучшения промтов и SOS-подсказок
- Не требует отдельной модели — просто SQL-агрегация по record_links

### 15.4 Обучение мини-модели (Фаза далёкого будущего)

- Только когда есть 5 000+ обезличенных ответов
- Цель: не советы, а сжатие смыслов, определение фреймов, подбор примеров
- Модель: Mistral / Hermes, файнтюн на корпусе ответов + литература АА/АН
- Включение в Helper Prompt как «слой опыта коллектива»

### 15.5 Расширение на «обычных людей»

- Архитектура 9 блоков — универсальная, не только для зависимых
- Потенциальные аудитории: люди в терапии, выгорание, созависимые (Ал-Анон)
- Блок «Восстановление» можно отключить → получается универсальный дневник самопознания
- Реализовывать после стабилизации основного продукта

## 16. РОЛЬ «СПОНСОР» — СПЕЦИФИКАЦИЯ

### 16.1 Что такое спонсор в 12 шагах

Спонсор — опытный участник программы, который ведёт подопечного по шагам.
Это НЕ терапевт. Это человек с личным опытом выздоровления.

### 16.2 MVP спонсора (FREE — реализовать первым)

**Связь спонсор ↔ подопечный:**

- Пользователь вводит Telegram username спонсора → запрос
- Спонсор подтверждает/отклоняет
- Один спонсор — несколько подопечных
- Подопечный может отвязаться в любой момент

**Просмотр прогресса подопечного:**

- На каком шаге, сколько вопросов пройдено
- Факт ответа (дата/время), БЕЗ текста ответа
- Последняя активность (когда был в боте)

**Уведомления спонсору (базовые):**

- Подопечный начал новый шаг
- Подопечный застрял (3+ дня без ответа)
- Подопечный нажал SOS

**Техническая реализация:**

- Роль `sponsor` в UserRole enum (уже есть в БД)
- Таблица `sponsor_relationships` (sponsor_id, mentee_id, status, created_at)
- Команда `/sponsor_panel` или кнопка «Мои подопечные»

### 16.3 Расширенный спонсор (Premium — Фаза 2)

- Лента активности подопечного (действия без текстов)
- Быстрые шаблоны ответов: «Хорошо идёшь», «Перечитай вводную», «Подумай глубже»
- Кнопка «Обратиться к спонсору» у подопечного → сообщение спонсору по текущему вопросу
- Статусы: «Активен», «В паузе», «Не отвечает»

### 16.4 Аналитика для спонсора (PRO — Фаза 3)

- Краткие summary по подопечному (повторяющиеся фреймы, эмоции)
- Сессия обратной связи после завершения шага
- Подопечный управляет видимостью: может скрыть тексты, показывая только статус

### 16.5 Правила приватности (КРИТИЧНО)

- Спонсор НИКОГДА не видит тексты ответов без явного согласия подопечного
- По умолчанию: только факт ответа + дата + номер шага/вопроса
- Подопечный может включить «Показывать тексты спонсору» в настройках
- Все просмотры логируются (access_logs)
